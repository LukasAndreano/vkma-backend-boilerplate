# 👨🏻‍💻 Стартовый кит бэкенда на Express + TS (no strict)
### ❓Что это такое ❓
Это стартовый кит _(шаблон)_ для создания бэкенда на платформе VK Mini Apps.

Используемые в ките технологии / библиотеки / фреймворки:
- [Express](https://expressjs.com/)
- [TypeScript](https://www.typescriptlang.org/)
- [VKMA Params Checker (моё)](https://www.npmjs.com/package/vkminiapps-params-checker)

### ⚙️ Начало работы
Склонируйте репозиторий в нужную вам директорию:
```bash
git clone https://github.com/LukasAndreano/vkma-backend-boilerplate.git <name>
```
Установите зависимости:
```bash
cd <name>
npm i
```

После установки всех зависимостей Вам необходимо отредактировать файл **.env**, изменив доступы. Пояснение того, что там происходит:    

PORT - порт, на котором будет запущен сервер (по дефолту у меня 4000)  

DB_HOST - хост mysql (обычно localhost или 127.0.0.1) 
DB_PORT - порт, на котором запущен mysql (обычно 3306)  
DB_USER - пользователь mysql    
DB_NAME - название базы данных      
DB_PASSWORD - пароль пользователя mysql      

SERVICE_KEY - ключ доступа сервиса (защищённый ключ)      
AUTHORIZATION_LIFETIME - время жизни параметров запуска (по дефолту в моих сервисах - 7200 секунд). Можно установить значение **0** - без ограничений.

Теперь можно запускать локальный сервер:
```bash
npm run dev
```

Откройте браузер и убедитесь, что всё работает корректно:
```bash
http://localhost:4000
```

### 💡  Идея работы роутинга
Категории содержат в себе endpoint'ы с одним методом _(GET, POST, PATCH и тд)_ и соединяются в группу через index.ts. Далее все группы вытягиваются из routes.json.
> В будущем идея создания групп через index.ts и require будет удалена, вместо неё будет цикл, подключающий все файлы из routes/v1.
